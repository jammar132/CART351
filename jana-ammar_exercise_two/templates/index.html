Welcome to CART 351 Exercise Two

This Flask exercise demonstrates how to use Flask routes, HTML templates, and the Jinja templating engine.

What I Did

I built a Flask web application that includes multiple routes and templates. The /t1 page displays variables using Jinja — including lists, conditionals, and images — while the /t2 page demonstrates a form using the GET method that combines user input and replaces all vowels with asterisks.

Issues I Encountered

  • At first, my web server wasn’t running properly. I kept seeing an error that said Access to 127.0.0.1 was denied (HTTP 403).I didn’t immediately know whether the issue came from my Flask code, my local environment, or my browser.
  • I used ChatGPT to help me work through the debugging process step by step. ChatGPT guided me to test the connection through the terminal using the command:curl -i http://127.0.0.1:5000/
  • This showed that the request wasn’t actually reaching my Flask application, which helped confirm that the issue wasn’t caused by my code but rather something with how the local server was being accessed. ChatGPT walked me through multiple troubleshooting steps — trying different ports, disabling possible conflicts, running Flask without the reloader, and confirming that the app was set to app.run(debug=True).
  • Eventually the server launched properly and my project began loading as expected. Through this process I learned how to read terminal responses, interpret HTTP error codes, and identify whether an issue comes from the code or from the system environment. This debugging session also helped me understand the importance of checking the console output, server logs, and using tools like curl to trace what’s really happening

What was the error?

Technically — the problem was AirPlay using port 5000 on my system.

  • Flask normally starts its dev server on port 5000.
  • macOS Monterey and newer reserve that same port for AirPlay Receiver (AirTunes).
  • So when http://127.0.0.1:5000/ opened, the request never reached Flask — it was intercepted by AirPlay, which replied “403 Forbidden.”

Reflection

Through this exercise, I learned how Flask connects routes to templates, how GET data travels through the URL, and how small configuration issues (like port conflicts) can stop a server from running. I also learned how Jinja variables make it easy to reuse dynamic data in HTML templates. This project helped me strengthen both my Python and front-end debugging skills.
